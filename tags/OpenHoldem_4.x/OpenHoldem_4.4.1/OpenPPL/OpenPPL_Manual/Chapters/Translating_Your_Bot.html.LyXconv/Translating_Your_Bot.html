Translating Your Bot

Before you can use your bot written in OpenPPL with OpenHoldem you have to translate it to OH-script, Openholdems native programming language. This job gets done by the converter called “OpenPPL.exe”. It also adds the library of supporting functions (“OpenPPL_Library.ohf”)

Prerequisites

  • OpenPPL code, OpenPPL library and OpenPPL converter in the same directory.
  • Source-code needs to be a text file in plain ASCII-format. No Word-document, 16 bit-encodings or other weird formats.

Translation Steps

  1. Open the command prompt (DOS-box).
  2. Change to the directory where your bot is located. Use the cd command for that.
  3. Type at the command line
    OpenPPL.exe my_source_code.txt > my_translated_bot.ohf

    In case you are not experienced dealing with the console we explain this step in detail:

    1. “OpenPPL.exe” is the translator that gets executed
    2. “my_source_code.txt” has to be the real name of your bot source of course. It is given as a command line argument to the converter.
    3. Usually the generated output gets to the console (which is funny to see). However with the “>” sign you can redirect it to a file.
    4. “my_translated_bot.ohf” finally is the converted source code after successful translation. In case of an error the file is incomplete and unusable, but does not get deleted for better problem diagnostics.
  4. After that you have an OH-script file that you can with load with OpenHoldem. Give it a try with PokerAcademy, PokerTH or ManualMode and you will see, that it will play something that looks like poker.

General tips

Working with the command-line (DOS-prompt) may be unaccustomed to some younger people who got born after the DOS-age and who don’t know Linux either. But once you get used to it it is simpler than a GUI and working with a mouse. Two tips to simplify your task:

  • You don’t have to type everything at the command-line. Use the tab-key for auto-completion of filenames.
  • Once you have entered the correct command you don’t have to input it again (in case of syntax errors). With arrow up and arrow down you can browse the history of old commands.
